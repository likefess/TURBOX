#  Оплата задолженности
___________________________________________________________________________________________________________________________

**Назначение:**  Документ предоставляет возможность ввода информации о платежах, проведенных (или полученных) на предприятии в рамках оплаты возникших обязательств
(по начисленной задолженности).
Возможно сформировать данные как по авансовым платежам, проведенным или полученным на предприятии в рамках погашения возникшего обязательства, так и по платежам,
не являющимися авансовыми (окончательный расчет и т.п.).

**Путь в системе:** Расчеты с субъектами / Документы / Документы / Оплата задолженности

**Порядок работы:**

1. Открыть картотеку "Оплата задолженности" и создать новую запись по кнопке *Добавить* ![](topic:Com.AddFiles.Btn_Add.png);

    ![](topic:.AddFiles.Screenshot_20250.jpg)

2. Заполнить основные данные документа:

    * **Номер** - номер документа. По умолчанию автоматически указывается номер согласно нумератора в формате NNNNNNN/ГГГГ, где NNNNNNN – порядковый номер, ГГГГ – номер года.
    Допускается корректировка пользователем, с учетом наличия уникальности;
    * **Дата** - дата документа. Дата ввода данных в систему. По умолчанию ставится текущая системная дата;
    * **Предприятие** - наименование нашего предприятия, по которому оформляется оплата задолженности (обязательства). Выбор из списка Наших предприятий, имеющихся в системе.
    Карточка выбранного предприятия доступна по кнопке ***Переход*** ![](topic:Com.AddFiles.Btn_go.png);
    * **Наименование** - наименование документа. По умолчанию указывается номер, дата, тип начисления и валюта документа. Допускается корректировка пользователем;
    * **флаг** ***Замок*** - в установленном состоянии означает, что документ закрыт для редактирования;
    * **флаг** ***Завершен*** - в установленном состоянии означает, что документ закрыт по учету;
    * **Кнопка** ***Проводить***  - при нажатии кнопки, документ становится проведенным в учете, по нему формируются оперативные проводки. При проведении документа автоматически
    производится переоценка суммы оплаты (в валюте, отличной от базовой валюты) и формируется документ [Переоценка задолженности](topic:.Расчеты с субъектами.Документы.Документы.Переоценка задолженности).
    При отмене проведения документа (кнопка ***Отменить***), все сформированные по данному документу оперативные проводки и документ переоценки задолженности удаляются из системы.
    При проведении документа предусмотрен контроль:
        - по сумме оплаты: сумма оплаты не должна превышать сумму начисления, с учетом ранее проведенных оплат, взаимозачетов и переоценок;
        - по соответствию Типа начисления: тип начисления, указанный в документе «Начисление задолженности» должен соответствовать типу начисления, указанному в
        документе «Оплата задолженности».
    Если контроль не пройден, то выводится сообщение об ошибке. Документ при этом сохраняется, но не проводится.

3. Заполнить поля на закладке **Документ**:

    * **Тип начисления** - указывается тип начисления, по которому оформляется оплата задолженности (обязательства). Выбор вручную из выпадающего списка.
    Значение по умолчанию = "Расчеты с поставщиками";
    * **Контрагент** - указывается наименование контрагента (юридического или физического лица), являющегося второй стороной в данном обязательстве (задолженности).
    Выбор из справочника «Контрагенты» (кнопка Выбор ![](topic:Com.AddFiles.Btn_select.png)). Карточка выбранного предприятия доступна по кнопке
    ***Переход*** ![](topic:Com.AddFiles.Btn_go.png);
    * **Договор** - договор, согласно которого возникла задолженность (обязательство), по которой проводится оплата. Выбор из справочника «Договор»
    (кнопка Выбор ![](topic:Com.AddFiles.Btn_select.png)), с предустановленным фильтром по параметрам Наше предприятие, Контрагент.
    Поле отображается только при Тип начисления = Расчеты с поставщиками, Расчеты с покупателями, Расчеты с прочими дебиторами, Расчеты с прочими кредиторами.
    Обязательно к заполнению при Тип начисления = Расчеты с поставщиками и Расчеты с покупателями. Карточка выбранного договора доступна по кнопке ***Переход*** ![](topic:Com.AddFiles.Btn_go.png);
    * **Тип договора** - указывается признак договора: Приходный или Расходный. Заполняется автоматически согласно карточки Договор;
    * **Налог** - указывается налог, по которому производится оплата задолженности (обязательства). Выбор из справочника «КБК» (кнопка Выбор ![](topic:Com.AddFiles.Btn_select.png)).
    Доступен выбор записи или группы.
    Поле доступно только при Тип назначения = Расчеты по налогам;
    * **Дата оплаты** - дата фактической оплаты, выбор вручную из календаря. Заполняется автоматически текущей системной датой;
    * **Тип оплаты** - указывается тип проводимого платежа. Выбор из списка : Приход / Расход;
    * **флаг** ***Аванс*** - флаг устанавливается при условии, что данный платеж является авансовым платежом;
    * **Платежный документ** - наименование, номер и дата платежного документа, по которому проведен платеж. Заполняется пользователем вручную;
    * **Документ обязат-ва** - указывается документ [Начисление задолженности](topic:.Расчеты с субъектами.Документы.Документы.Начисление задолженности),
    по которому проводится оплата.
    При установленном флаге *Аванс* поле не обязательное для заполнения.
    * **Примечание** - примечания пользователя. Заполняется вручную;
    * **Сумма по документу** - сумма оплаты. Сумма с учетом НДС (при наличии) платежного документа, согласно которого проведена оплата задолженности. Заполняется вручную;
    * **Валюта документа** - наименование валюты задолженности. Выбор из перечня валют, по которым ведется учет на предприятии (согласно данных справочника «Валюты»).
    Автоматически заполняется из документа Договор или Документ обязательства, если документ указан. По кнопке ***Переход*** ![](topic:Com.AddFiles.Btn_go.png)
    производится переход на форму карточки «Валюта». Указать валюту оплаты, отличную от базовой (операционной) валюты возможно только при условии наличия заполненного поля «Договор».
    При наличии заполненного поля «Договор» Валюта документа всегда равна Валюте договора. Осуществляется проверка на соответствие валюты документа оплаты валюте
    документа обязательства. В случае несоответствия дополнительно выводится сумма в валюте задолженности, рассчитанной по кросс-курсу обмена 2-х валют;
    * **Курс** - указывается курс выбранной валюты документа на дату оплаты. Курс указывается согласно истории изменения курса, указанной в карточке валюты. Поле отображается
    только при условии, что Валюта документа не равна Базовой валюте. Допускается ручное редактирование курса пересчета пользователем. По кнопке
    ***Обновить*** ![](topic:Com.AddFiles.Buttons.Btn_Refresh_mini.png) производится обновление значения курса валюты;
    * **в валюте задолжен.** - рассчитывается значение суммы в валюте задолженности по кросс-курсу обмена валют;
    * **Валюта задолженности** - наименование валюты задолженности, соответствующей валюте указанного договора. Заполняется автоматически, не редактируется;
    * **Курс** - указывается курс валюты задолженности на дату оплаты. Курс указывается согласно истории изменения курса, указанной в карточке валюты. Допускается ручное
    редактирование курса пересчета пользователем. По кнопке ***Обновить*** ![](topic:Com.AddFiles.Buttons.Btn_Refresh_mini.png) производится обновление значения курса валюты;
    * **Сумма в опер. валюте** - сумма оплаты в операционной валюте. Предусмотрен автоматический расчет суммы в операционной валюте по формуле: *Сумма по документу * Курс*.
    Допускается ручная корректировка значения пользователем. Поле отображается только при условии, что Валюта документа не равна Базовой валюте;
    * **Опер. валюта** - наименование операционной валюты (базовой валюты). Указывается автоматически согласно данных документа «Настройка учетной политики»;
    * **блок Создал, изменил** - в полях блока отображается логин пользователя, создавшего и внесшего изменения в документ, а также дата и время создания и изменения документа.
    Указывается наименование пользователя, создавшего и внесшего изменения в документ, а также дата и время создания и изменения документа.

4. Провести документ по кнопке ![](topic:Com.AddFiles.Buttons.Btn_Provodit.png).

5. Закладка **Проводки**

    На закладке формируется отчет по проводкам документа. Отображается на форме только в проведенном документе.
    Отчет по проводкам содержит перечень проводок по оперативным счетам, сформированных по данным документа.
    На закладке указан перечень параметров отчета, которые показываются в отчете. Вывод на экран параметра отчета производится посредством установки флага в данном параметре.

    Формирование и обновление отчета производится по кнопке *Обновить* ![](topic:Com.AddFiles.Buttons.Btn_Refresh_mini.png).

    ![](topic:.AddFiles.Screenshot_20251.jpg)

6. Закладка **Связанные документы**

    Отображается перечень документов, сформированных на основании данного документа оплаты ( документы [Зачет аванса](topic:.Расчеты с субъектами.Документы.Документы.Зачет аванса), [Переоценка задолженности](topic:.Расчеты с субъектами.Документы.Документы.Переоценка задолженности)).

    Переход на форму связанного документа осуществляется по кнопке ***Переход*** ![](topic:Com.AddFiles.Buttons.Btn_togo2.png).

    ![](topic:.AddFiles.Screenshot_20252.jpg)

    Отображается перечень документов, сформированных на основании данного документа оплаты ( документы [Зачет аванса](topic:.Расчеты с субъектами.Документы.Документы.Зачет аванса), [Переоценка задолженности](topic:.Расчеты с субъектами.Документы.Документы.Переоценка задолженности)).

    Переход на форму связанного документа осуществляется по кнопке ***Переход*** ![](topic:Com.AddFiles.Buttons.Btn_togo2.png).

7. Закладка **Вложения**

На закладке отображаются все прикрепленные документы/изображения, относящиеся к данному документу и хранящиеся в файловом архиве. Добавить документы/изображение можно
с помощью кнопок *Добавить файлы*/*Изображение* соответственно.


**Примечание:** При подключенном модуле «Казначейство» системы ТУРБО предоставляется возможность загрузки данных по оплатам, проведенным в системе.
Интеграция данных модулей «Расчеты с субъектами» (документ «Оплата задолженности») и «Казначейство» (документ «Движение по банку») производится только при условии наличия
проведенного по Нашему предприятию документа "Интеграция данных по расчетам".

В системе производится автоматическое формирование документа «Оплата задолженности», при использовании сервиса «Разнести данные» в документе «Движение по банку».
Настройка интеграции данных описана в документе [Интеграция данных по расчетам с субъектами](topic:AppBase.AppBase.Настройки.Интеграция данных).
