#Типы объектов (процессов)

Настройка карточки типа объекта позволяет задать параметры, характеризующие настраиваемый тип. 

**Путь:** Студия компонентов/Структура/Типы объектов

**Порядок работы с формой:**

Для добавления нового типа объекта (процесса) в картотеку нажмите кнопку **Добавить** ![](topic:Com.AddFiles.Btn_Add.png), для редактирования существующего кнопка **Редактировать**![](topic:Com.AddFiles.Btn_Edit.png):



1.В верхней части формы отображаются элементы для описания основных свойств типа объекта:

- Поле Название - заполняется вручную наименование типа объекта (процесса);
- Поле Код - уникальный код, заполняется автоматически, редактируется вручную. Новый код не должен совпадать ни с одним из ранее созданных;
* Комментарий - при необходимости внести произвольный комментарий.

Остальные настройки определены по разделам карточки, в каждом из разделов настраиваются соответствующие настройки.

2.Основное

2.1 Классификация:

- Mtl-класс объекта заполняется из списка Mtl-класс объекта (кнопка Выбор ![](topic:Com.AddFiles.Btn_select.png));
* Rt- класс объекта - заполняется из списка Rt- класс объекта (кнопка Выбор ![](topic:Com.AddFiles.Btn_select.png));
- Версия настройки - вручную.

2.2 Коды

Поля данного раздела заполняются вручную в соответствии со свойствами данного объекта.

3.Внешний вид

В разделе "Внешний вид" карточки типа объекта настраиваются параметры, касающиеся внешнего вида типа объекта.
В разделе перечислены следующие опции: 

3.1 Список представлений

Таблица позволяет определить состав представлений, которые будет содержать данный тип объекта. Для перехода к форме Представление объекта нажмите на кнопку в таблице ![](topic:Com.AddFiles.Buttons.Btn_unloading.png):

* В поле "Представление" - указывается представление, которое будет содержаться в данном типе объекта. Представление выбирается из картотеки "Представления объекта", которая открывается по нажатию на кнопку Выбор![](topic:Integration.AddFiles.Buttons.Btn_select.png). 
* В поле "На закладке" - указывается наименование представления, которое будет отображено на закладке объекта, если представлений в объекте несколько. По умолчанию, отображается наименование, заданное в настройках представления. 
* Поле "Подсказка" служит для ввода подсказки к представлению типа объекта.
* Флаг "Использовать при создании нового документа" отвечает за использование представления при создании нового документа. Если данный флаг установлен, то представление, в строке которого установили флаг, будет использоваться при создании нового документа. Если флаг не установлен, то представление использоваться не будет. 
* Флаг "Только для чтения" отвечает за отображение представления, которое будет доступно только на чтение. Другими словами, редактировать что-либо в представлении не будет возможным, если флаг установлен. 
* Воспользуйтесь кнопкой Сервис ![](topic:Com.AddFiles.Buttons.Btn_Services.png):
     * "Добавить Условие" - открывшееся поле "при условии" служит для ввода условия, от выполнения которого зависит отображение представления. Условие - это реквизит объекта, желательно, чтобы это был реквизит-лимит.
     * "Добавить комментарий" - в открывшемся поле Комментарий внести произвольное пояснение.

Строки в таблицу  добавляются/удаляются с помощью опции Добавить кадр/Удалить кадр в меню вызываемом по щелчку правой кнопки мыши или с помощью клавиш Ins/Del.


3.2 Дополнительные настройки внешнего вида.

Следующие настройки позволяют указать реквизиты для записи значений периода карточки. Период в карточке типа объекта является глобальным, т.е. действует для всей карточки, и отображается только на представлениях, для которых период имеет смысл (например, тип представления "Список объектов"). Однако каждое такое представление может иметь свой период, отличный от всей карточки. 

* Настройка "Верхнее представление" служит для выбора представления типа "Документ", позволяющее указать реквизиты, которые должны быть видны во всей карточке типа объекта. Например, верхнее представление может содержать реквизит, отображающий дату создания типа объекта, либо реквизит нумератор. 
* Установка флагов напротив настройки "Видимость кнопок" позволяет отобразить, соответствующую кнопку в верхнем фрейме карточки типа объекта. 
* Реквизит "Период". В данном поле указывается реквизит, который будет выступать в качестве значения периода. Тип значения указанного реквизита должен быть целочисленным (так как каждое значение периода времени зарегистрировано в системе, как целочисленная константа). 
Например, зарегистрируем в системе реквизит "Период" с типом значения целое. Свяжем его с определенным целочисленным дополнительным полем. Укажем данный реквизит в качестве значения периода. Тогда, при смене значения периода в карточке, его значение будет записано в настроенный реквизит "Период". 
* Реквизит "Дата начала". В данном поле необходимо указать реквизит, в который будет записываться значение даты начала периода при его смене в карточке объекта. Типом значения указанного реквизита должна быть дата. 
* Реквизит "Дата окончания". Опция позволяет задать реквизит, в который будет записываться значение даты конца периода. Типом значения реквизита необходимо чтобы была дата. Выбрать нужный реквизит можно в картотеке, либо вписав его вручную в настраиваемое поле. 
* Описание объекта заполняется с помощью выпадающего списка: По умолчанию\Код и дата\Наименование\Код (номер)\Тип и код\ Текущее представление\Код и текущее представление.



4.Пересчеты

4.1 В разделе "Автоматические пересчеты"  настраиваются формулы, которые пересчитываются автоматически каждый раз, когда меняется значение одного из реквизитов, участвующих в них. Это наиболее простой и очевидный способ указания взаимозависимостей между значениями реквизитов. Однако обратите внимание, что настройка автоматического пересчета не означает, что значение реквизита не может быть вычислено по другой формуле или изменено вручную. Указание формулы в разделе автоматических пересчетов всего лишь означает, что контроль зависимостей значения данного реквизита от других реквизитов отдается системе, а события, при которых она перевычисляется, не описываются вручную, как это делается при "Пересчеты при изменении значений", "Пересчеты по событиям". 

Автоматические пересчеты задаются в таблице. Нажмите на кнопку в таблице ![](topic:Com.AddFiles.Buttons.Btn_unloading.png): 

* В поле "Вычислить" указывается реквизит, смена значения которого будет вызывать пересчет по заданной формуле. Кнопка Выбор ![](topic:Com.AddFiles.Buttons.Btn_select.png) в поле открывает картотеку "Реквизиты объекта". 
* В поле "по формуле" указывается формула, по которой будет происходить пересчет. Формула задается аналогично [Редактор формул](topic:.Студия компонентов (машина реквизитов).Редактор формул). 

Воспользуйтесь кнопкой Сервис ![](topic:Com.AddFiles.Buttons.Btn_Services.png), которая открывает дополнительные поля:
* "Добавить Условие" - в открывшемся поле "при условии" задается логическое условие. Если оно выполняется, то при смене значения заданного реквизита будет происходить пересчет указанной формулы. Иначе пересчет не будет производиться.
       * Само условие, а также используемые в нем реквизиты и записи, вводятся в строке "при условии".
       * Кнопка "Реквизит" ![](topic:Com.AddFiles.Buttons.Btn_Settings_menager.png) позволяет указать реквизит. Выбранный реквизит будет отображен в первой строке редактора под соответствующим порядковым номером.
       * Кнопка "Запись" ![](topic:Com.AddFiles.Buttons.Btn_base.png) позволяет выбрать необходимую запись из представленных классов записи. Выбранная запись, так же как и реквизит, будет отображена в строке редактора под соответствующим порядковым номером.
       * Кнопка "Проверка" ![](topic:Com.AddFiles.Buttons.Btn_zagr.png) позволяет проверить условие на синтаксические ошибки.
* "Добавить комментарий" - в открывшемся поле Комментарий внести произвольное пояснение.

В таблице установка флага "Выкл" позволяет отключить пересчет реквизита, в строке которого установили флаг. 

Строки в таблицу  добавляются/удаляются с помощью опции Добавить кадр/Удалить кадр в меню вызываемом по щелчку правой кнопки мыши или с помощью клавиш Ins/Del.


4.2 В разделе "Пересчеты при изменении значений" указывается, какие пересчеты следует запускать при изменении значений определенных реквизитов в настраиваемом типе объекта. 

Каждый пересчет задается настройками в таблице.  Нажмите на кнопку в таблице ![](topic:Com.AddFiles.Buttons.Btn_unloading.png):

* В поле "При изменении" указывается реквизит, значение которого изменилось. По нажатию на кнопку Выбор ![](topic:Com.AddFiles.Buttons.Btn_select.png) в поле, открывается картотека "Реквизитов объекта", откуда можно выбрать необходимый реквизит. 
* В поле "Вычислить" указывается реквизит, значение которого необходимо изменить по заданной формуле. По нажатию кнопки Выбор ![](topic:Com.AddFiles.Buttons.Btn_select.png) в данном поле открывается картотека "Реквизитов объекта". 
* Флаг "Пересчитывать всегда" отвечает за выполнение постоянного пересчета. Постоянные пересчеты исполняются при любом изменении реквизита, независимо от того, был ли он изменен вручную или в результате работы другого пересчета. Если пересчет не является постоянным, он запускается только после непосредственного изменения значения реквизита - вручную или сервисной процедурой. 

Воспользуйтесь кнопкой Сервис ![](topic:Com.AddFiles.Buttons.Btn_Services.png), которая открывает дополнительные поля:
* "Добавить Условие" - в открывшемся поле "при условии" задается логическое условие. Если оно выполняется, то при смене значения заданного реквизита будет происходить пересчет указанной формулы. Иначе пересчет не будет производиться.
       * Само условие, а также используемые в нем реквизиты и записи, вводятся в строке "при условии".
       * Кнопка "Реквизит" ![](topic:Com.AddFiles.Buttons.Btn_Settings_menager.png) позволяет указать реквизит. Выбранный реквизит будет отображен в первой строке редактора под соответствующим порядковым номером.
       * Кнопка "Запись" ![](topic:Com.AddFiles.Buttons.Btn_base.png) позволяет выбрать необходимую запись из представленных классов записи. Выбранная запись, так же как и реквизит, будет отображена в строке редактора под соответствующим порядковым номером.
       * Кнопка "Проверка" ![](topic:Com.AddFiles.Buttons.Btn_zagr.png) позволяет проверить условие на синтаксические ошибки.
* "Добавить комментарий" - в открывшемся поле Комментарий внести произвольное пояснение.

В таблице установка флага "Выкл" позволяет отключить пересчет реквизита, в строке которого установили флаг. 

Строки в таблицу  добавляются/удаляются с помощью опции Добавить кадр/Удалить кадр в меню вызываемом по щелчку правой кнопки мыши или с помощью клавиш Ins/Del.


4.3 Пересчеты в разделе "Пересчеты по событиям"  аналогичны пересчетам при изменении значений с той разницей, что пересчет запускается не при изменении значения определенного реквизита, а при наступлении одного из заранее определенных событий объекта, таких как "создание нового объекта", "ввод новой позиции", "любое сохранение" и т.п. 

Каждый пересчет задается настройками в таблице.  Нажмите на кнопку в таблице ![](topic:Com.AddFiles.Buttons.Btn_unloading.png):

* В поле "По событию" указывается некоторое событие, при наступлении которого запускается пересчет. Событие выбирается с помощью выпадающего списка. 
* В поле "Заполнить" указывается реквизит, который необходимо заполнить при наступлении определенного события. По нажатию на кнопку Выбор ![](topic:Com.AddFiles.Buttons.Btn_select.png) в данном поле открывается картотека "Реквизитов объекта". В зависимости от события, картотека будет отфильтрована по шапочным или по табличным реквизитам. 


В таблице установка флага "Выкл" позволяет отключить пересчет реквизита, в строке которого установили флаг. 

Строки в таблицу  добавляются/удаляются с помощью опции Добавить кадр/Удалить кадр в меню вызываемом по щелчку правой кнопки мыши или с помощью клавиш Ins/Del.


5.Видимость/доступность

В разделе "Видимость/Доступность" карточки типа объекта настраиваются доступность и видимость реквизитов, таблиц и представлений в данном типе объекта. 

5.1 В разделе "Реквизиты" устанавливаются надписи, доступность, либо отображение определенного реквизита в настраиваемом типе объекта. 

Нажмите на кнопку в таблице ![](topic:Com.AddFiles.Buttons.Btn_unloading.png):

* Поле "Установить" служит для выбора действия, которое необходимо установить у определенного реквизита. Действие, совершаемое над реквизитом, выбирается из раскрывающегося списка. В списке представлены варианты: 
    * Доступность - будет настраиваться доступ к реквизиту в данном типе объекта; 
    * Видимость - настраивается отображение реквизита в настраиваемом типе объекта; 
    * Надпись - устанавливается надпись реквизита, которая будет отображена в данном типе объекта. 
* В поле "У реквизита" указывается реквизит объекта, для которого будут устанавливаться доступность, видимость, либо надпись. Реквизит выбирается из картотеки "Реквизиты объекта" по нажатию на кнопку Выбор ![](topic:Com.AddFiles.Buttons.Btn_select.png).
* Настройка "По формуле" служит для ввода формулы, от выполнения которой зависит установка видимости, доступности, либо надписи для реквизита. Формула задается аналогично "[Редактор формул](topic:.Студия компонентов (машина реквизитов).Редактор формул)".

Воспользуйтесь кнопкой Сервис ![](topic:Com.AddFiles.Buttons.Btn_Services.png), которая открывает дополнительные поля:
* "Добавить Условие" - в открывшемся поле "при условии" задается логическое условие.
       * Само условие, а также используемые в нем реквизиты и записи, вводятся в строке "при условии".
       * Кнопка "Реквизит" ![](topic:Com.AddFiles.Buttons.Btn_Settings_menager.png) позволяет указать реквизит. Выбранный реквизит будет отображен в первой строке редактора под соответствующим порядковым номером.
       * Кнопка "Запись" ![](topic:Com.AddFiles.Buttons.Btn_base.png) позволяет выбрать необходимую запись из представленных классов записи. Выбранная запись, так же как и реквизит, будет отображена в строке редактора под соответствующим порядковым номером.
       * Кнопка "Проверка" ![](topic:Com.AddFiles.Buttons.Btn_zagr.png) позволяет проверить условие на синтаксические ошибки.
* "Добавить комментарий" - в открывшемся поле Комментарий внести произвольное пояснение.

В таблице установка флага "Выкл" служит для временного отключения строки в таблице. . 

Строки в таблицу  добавляются/удаляются с помощью опции Добавить кадр/Удалить кадр в меню вызываемом по щелчку правой кнопки мыши или с помощью клавиш Ins/Del.

5.2 В разделе "Таблицы" настраиваются видимость и доступность типов учетного движения для данного типа объекта. 

Настройка представлена в виде таблицы аналогична настройке видимости "Реквизиты", отличие в следующих параметрах:
 
* Поле "Установить" служит для выбора действия, которое необходимо установить у определенной подтаблицы. Действие, совершаемое над подтаблицей, выбирается из раскрывающегося списка. В списке представлены варианты: 
      * Доступность - будет настраиваться доступ к таблице в данном типе объекта; 
      * Видимость - настраивается отображение таблицы в настраиваемом типе объекта; 
* В поле "У таблицы" указывается тип движения объекта, для которого будут устанавливаться доступность или видимость. Тип движения выбирается из картотеки "Типы движений", по нажатию на кнопку Выбор ![](topic:Com.AddFiles.Buttons.Btn_select.png).

В таблице установка флага "Выкл" служит для временного отключения строки в таблице. . 

Строки в таблицу  добавляются/удаляются с помощью опции Добавить кадр/Удалить кадр в меню вызываемом по щелчку правой кнопки мыши или с помощью клавиш Ins/Del.



5.3 В разделе "Представления" раздела настраиваются видимость и доступность представлений для данного типа объекта. 

Настройка представлена в виде таблицы аналогична настройке видимости "Реквизиты", отличие в следующих параметрах:

* Поле "Установить" служит для выбора действия, которое необходимо установить у определенного представления. Действие, совершаемое над представлением, выбирается из раскрывающегося списка. В списке представлены варианты: 
      * Доступность - будет настраиваться доступ к представлению в данном типе объекта; 
      * Видимость - настраивается отображение представления в настраиваемом типе объекта;

6.Общие реквизиты

В разделе "Общие реквизиты" настраиваются отношения обобщения между реквизитами, такие как "является общим", "является суммой", "является умолчанием". Как правило, отношения обобщения между табличным реквизитом и реквизитом шапки могут быть настроены прямо в карточке реквизита. Однако какие-то отношения являются локальными, специфическими для конкретного типа объекта, и поэтому их имеет смысл настраивать в данном разделе карточки типа объекта. Каждое обобщение настраивается следующими опциями в форме Общий реквизит  по кнопке  ![](topic:Com.AddFiles.Buttons.Btn_unloading.png): 

* В поле "Реквизит шапки" указывается шапочный реквизит объекта, который будет являться общим/умолчанием/суммой для табличного реквизита. По нажатию кнопки Выбор ![](topic:Com.AddFiles.Buttons.Btn_select.png) в данном поле открывается картотека "Реквизитов объекта", отфильтрованная по шапочным реквизитам. 
* В поле "Тип общности" выбирается вариант обобщения реквизита с помощью раскрывающегося списка: является Общим для, является Умолчанием для, является Суммой для.
* В поле "Табличный реквизит" указывается реквизит объекта, использующийся в подтаблице, для которого настраивается отношение обобщения. Реквизит выбирается из картотеки "Реквизитов объекта" по нажатию на кнопку Выбор![](topic:Com.AddFiles.Buttons.Btn_select.png). 
* В поле "Тип движения" выбирается тип учетного движения, в котором будет действовать отношение обобщения. Тип движения выбирается из картотеки "Типов движений" по нажатию на кнопку Выбор![](topic:Com.AddFiles.Buttons.Btn_select.png) в данном поле.

7.Ограничения

С помощью таких фильтров, например, можно ограничить выбор субъектов по их ролям или разрешить выбирать в реквизит-объект только объекты определенного типа. 

Ограничения задаются при помощи следующих настроек в форме Ограничение (кнопка ![](topic:Com.AddFiles.Buttons.Btn_setting.png)):

* В поле "Наложить ограничение" указывается формула, которая будет являться устанавливаемым фильтром на картотеку ссылочного реквизита.  Ограничение представляет собой формулу, возвращающую выражение строкового типа на языке фильтров "ТБ.Студии".

Настройка группы позволяет указать группу (папку), на которую будет установлен курсор в момент открытия картотеки ссылочного реквизита. 9.Библиотеки

В разделе "Библиотеки" указывается список ссылок на другие типы объектов, используемых в данном случае как источник бизнес-логики. Например, если в нескольких типах объектов используется один и тот же распространенный алгоритм взаимосвязей и пересчета группы реквизитов, то не требуется повторять его настройку в каждом из типов объектов. Достаточно настроить эти правила один раз в типе объекта, который впоследствии будет использоваться как библиотека. Обратите внимание, что настройки не копируются, а используются непосредственно из библиотек. Таким образом, если внести изменение в настройки библиотеки, то эти изменения сразу же будут применены ко всем типам объектов, где данная библиотека используется. 

Библиотеки образуют иерархию - при настройке библиотеки можно использовать ссылки на другие, более мелкие библиотеки. В конечном счете, в конкретном типе объекта появится функциональность из всех библиотек, которые были использованы при его настройке. 

Библиотеки к данному типу объекта добавляются при помощи таблицы. Таблица состоит из следующих настроек:
 
* В поле указываются типы объектов, являющиеся библиотеками, которые будут использоваться в настраиваемом типе объекта. Библиотеки выбираются при помощи кнопки Выбор ![](topic:Com.AddFiles.Btn_select.png) в поле, после нажатия которой открывается картотека "Типы объектов". 
* При помощи флага "Выкл" можно временно отключить использование библиотеки, в строке которой устанавливают флаг. 

Строки в таблицу  добавляются/удаляются с помощью опции Добавить кадр/Удалить кадр в меню вызываемом по щелчку правой кнопки мыши или с помощью клавиш Ins/Del.

***Примечание:*** *Состав и структура библиотек остается на усмотрение настройщика. Однако очевидно, что при разработке структуры бизнеса имеет смысл уделить особое внимание формализации часто встречающихся в объекте настройки случаев и выделению их в иерархию библиотек - это позволит упростить настройку конкретных типов объектов, а также внесение изменений в настройку.*
